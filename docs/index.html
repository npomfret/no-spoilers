No Spoilers

The F1 race weekend schedule for your Mac menu bar — with a hard guarantee that results never appear.

Install with Homebrew
brew install --cask npomfret/tap/no-spoilers
macOS Sequoia 15+  ·  Updates via brew upgrade

What you get

🏎

Full weekend at a glance

FP1 through the race — every session, its local start time, and a live countdown to the next one.

🔕

Spoiler-proof by design

Results are not hidden behind a toggle. The app's data model has no result fields — they simply don't exist.

📍

Always in your menu bar

Click the icon in the macOS menu bar to see the current race weekend. One click, no context switching.

⚡️

Native and lightweight

Written in Swift, uses only Apple frameworks. No Electron, no background daemons, no telemetry.

How it works

1

Install once

Run the Homebrew command above. The app lands in your Applications folder and adds itself to your menu bar.

2

Click the menu bar icon

A popover shows the current race weekend — all sessions, local times, and a countdown to the next live session. Sessions that have already ended are marked as ready to watch.

3

Watch on your schedule

Check whether a session has finished — safely — without any risk of stumbling across a result. Then watch whenever you're ready.

4

Stay up to date

Schedule data comes from the community-maintained f1calendar.com feed, updated each season. Run brew upgrade to get new app versions.

The spoiler guarantee

Most "spoiler-free" apps hide results behind a setting. No Spoilers goes further: the domain model has no result fields at all — not optional, not ignored, not hidden. The data source (f1calendar.com) is schedule-only by design. A Session has a name, a start time, and an end time. That's it. There is no code path that could ever surface a result, because the data to hold one doesn't exist anywhere in the app.
